    Software Life Cycle: Development Phase
  • Requirement analysis: to specify what services the proposed system will provide, to identify and conditions on those services, and to define how the outside world will interact with the system.
  • Design: Creating a plan for the construction of the proposed system.
  • Implementation: Involves the actual writing of programs, creations of data files, and development of databases.
  • Testing: Essantailly debugging and using the programs to fix errors and complete the final product.

    Modular Implementation Seperated the fuctionality of a program into independent, interchangeable modules.
    Unifed Modeling Language A more modern collection of tools that has been developed with the object-oriented paradigm in mind.
    Use Case Interactions between the system and its users in a use case diagram.
    Use Case Diagram A diagram that attempts to capture the image of the proposed system from the user's point of view.
    Data Relationships One-to-One: A relationship that has one entity is connected to a single entity. Eg: Type X is connected to Type Y. One-to-Many: A relationship that has one entity being connected to multiple other entities. Eg: Type X is connected to multiple Type Y enities. Many-to-Many: A relation ship has multiple entities connected to other multiple entities. Eg. Multiple Type X entities are connected to multiple Type Y entities.
    Design Pattern A predevoloped model for solving a recurring problem in software design.
    Software Quality Assurance Groups formed to oversee and enforce quality control system adopted by the organization.
    Software Testing Glass-box testing: is where a software tester is aware of the interior stucture of the software and uses this knowledge when designing the test. Black-box testing: Tests that do not rely on the software tester's knowledge and is tested by the user's point of view. Beta testing: Testing that falls into the black-box testing category. It's where a preliminary version of the software is given to a segment of the intended audience with the goal of learning how the software performs in real-life situations before the final version of the product. Alpha testing: Similar to beta testing, but it is performed at the developer's site.
    Documentation User documentation: explains the featurs of the software and describe how to use them. System documentation: Describes the software's internal composition so that the software can be maintained late in its life cycle. Techical documentation: Describes how a software system should be installed and serviced.
    Intellectual Property Law Software license: A legal agreement between the owner and user of a software product that grants the user certian permissions to use the product without transferring ownership rights. Eg:Buying a digital game is actually you buying a software license for the game. This is used in game consoles and big game platforms. Software Disclaimer: A legal statement that states the limitation of the license. Eg: warranties, limitation of liability, user responsibility. Best defenses against software liability claims: Disclaimer that state the limitations of their liability (out of their control). Eg: Nintedo warns and urges its users to not hack their consoles as it can damaged the system or brick it. It states that Nintendo is not liable for damages and will not replace the console if hacked.
